“Fear of the new”

3:00 – 5:00 · PROBLEM (2 min)

The barrier isn’t conceptual. It’s psychological.

Students who know:

  • What a linear model is
  • How to interpret coefficients
  • When to use a t-test

Freeze when they see:

  • model.fit(X, y) instead of lm(y ~ x)
  • pd.DataFrame instead of data.frame
  • := instead of <-

Same concept. Different costume.

  • This framing — “same concept, different costume” — is the through-line. Repeat it in the demo.
  • Mention groups disproportionately affected: career-switchers, first-gen students, students from teaching-focused programs without industry exposure.

Bad prompt

6:30 – 8:00 · DEMO part 1 (1.5 min)

Convert this R code to Python:

lm(mpg ~ wt + hp, data = mtcars)

You get: working code.

You don’t get: why ~ became a design matrix, why sklearn separates X and y, or that summary() has no direct equivalent.

  • Run this live. Show the output.
  • Pause on what’s MISSING: the conceptual map.
  • “This produces an answer. It doesn’t produce learning.”

Good prompt

8:00 – 10:00 · DEMO part 2 (2 min)

I'm an intro stats student fluent in R.
Translate this R code to Python, AND:

1. Add inline comments mapping each R line to its Python equivalent.
2. Flag any idiom differences (e.g., formula interface vs. design matrix).
3. End with one concept-check question I should be able to answer
   before I move on.

R code:
lm(mpg ~ wt + hp, data = mtcars)
  • Run this live. Show the output — it should include comments + a check question.
  • This is the centerpiece pattern. Name it: “learning goal + mapping + idiom flags + concept check.”
  • This pattern is in the GitHub repo as a template.

Python → R: same trick

10:00 – 11:30 · DEMO part 3 (1.5 min)

df.groupby("cyl")["mpg"].mean()

AI maps it to:

df |> group_by(cyl) |> summarize(mean_mpg = mean(mpg))

…and explains that method chaining ↔︎ the pipe.

  • This is the tidyverse cameo. Highlight that the pipe is the same idea as . chaining in pandas.
  • Students who learned dplyr first often “get” pandas faster once they see this.

30-sec SQL cameo

11:30 – 12:30 · DEMO part 4 (1 min)

SQL

SELECT cyl, AVG(mpg)
FROM mtcars
GROUP BY cyl
HAVING AVG(mpg) > 20;

dplyr

mtcars |>
  group_by(cyl) |>
  summarize(m = mean(mpg)) |>
  filter(m > 20)

One concept. Three costumes.

  • Move fast. Don’t explain SQL in depth — just show that GROUP BY/HAVING maps cleanly.
  • This is the moment the “same concept, different costume” line earns its keep.

Bridge vs. crutch

18:30 – 22:30 · GUARDRAILS (4 min)

Bridge

  • Annotate AI output
  • Translate, then close the tab and re-derive
  • Pair with a concept check
  • Scaffolded fading (week 1 → week 5)

Crutch

  • Copy-paste without reading
  • Skip the why
  • Never reproduce unaided
  • Same level of help every week
  • This is the most “Monday-morning useful” section. Slow down.
  • Each bullet is a real classroom move — give one quick example for each on the left.
  • “Scaffolded fading” = the key term they’ll Google later.

Who benefits most?

22:30 – 23:30 · WHO (1 min)

Benefits most:

  • Students switching ecosystems mid-program
  • Career-switchers with stats but new to programming
  • Students at teaching-focused institutions w/o industry exposure

At risk of over-reliance:

  • Students who haven’t built the underlying concept yet
  • Students using AI to skip — not to bridge
  • Be honest: AI translation is most useful AFTER the concept is solid in one language.
  • Don’t introduce it as a first encounter with linear regression.
